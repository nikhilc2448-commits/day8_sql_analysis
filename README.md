# Day 8 - SQL for Data Analysis

## Objective
This project demonstrates fundamental SQL concepts by creating a sample retail database and solving business-related analytical questions using SQL queries.

## Topics Covered
- CREATE DATABASE
- CREATE TABLE
- INSERT INTO
- SELECT
- WHERE
- ORDER BY
- LIMIT
- AND, OR, IN, BETWEEN, LIKE
- Aggregate Functions (COUNT, SUM, AVG, MIN, MAX)
- GROUP BY
- HAVING
- INNER JOIN
- LEFT JOIN
- RIGHT JOIN
- Subqueries
- CASE Statement

## Project Structure


## Dataset
A sample retail database containing:
- Customers
- Products
- Orders

## Business Questions Solved
1. Display all customer details.
2. Find customers from a specific city.
3. List the top 5 highest-value orders.
4. Calculate total sales by product category.
5. Find customers with total purchases above a threshold.
6. Display monthly sales.
7. Categorize products using CASE.
8. Retrieve customers with purchases above the average order value.
9. Display combined customer and order details using JOINs.
10. Generate sales insights using aggregate functions.

## SQL Concepts Demonstrated
- Data Definition Language (DDL)
- Data Manipulation Language (DML)
- Data Query Language (DQL)
- Filtering and Sorting
- Aggregate Functions
- Joins
- Grouping and Aggregation
- Subqueries
- Conditional Expressions (CASE)

## Key Insights
- Electronics generated the highest total sales.
- The top customer contributed the highest purchase amount.
- Monthly sales analysis identified the best-performing month.
- Product categories were classified using CASE statements.
- Aggregate analysis helps identify sales trends and customer behavior.
